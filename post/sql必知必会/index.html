《SQL必知必会》学习笔记#

第一章 了解SQL#

1.1 数据库的基础#

1.1.1 数据库#

数据库（database）：保存有组织的数据的容器。

DBMS：数据库管理系统

1.1.2 表#

表：某种特定类型数据的结构化清单。

表名：在同一数据库中不能使用相同的表名。

1.1.3 列和数据类型#

表由列组成

列（column）：表中的字段。

数据类型：用于限制列中所存储的数据类型

1.1.4 行#

行（row）：表中的记录

1.1.5 主键#

主键（primary key）：用于唯一表示表中每一行

主键的特点：

  • 唯一性
  • 不允许为NULL值
  • 不允许修改或更新
  • 不能重用（不能删除之后赋予新行）

1.2 什么是SQL#

SQL：Structured Query Language（结构化查询语言）

第二章 检索数据#

2.1 SELECT 语句#

关键字(keyword)：作为SQL组成部分的保留字，不能用作为表或列的名字。

SELECT检索数据，必须至少给出两条信息：选择什么，从什么地方选择

2.2 检索检索单个列#

SELECT prod_name FROM dbo.Products;

提示：

  • 建议SQL语句结尾总是加上分号
  • SQL语句不区分大小写
  • 在处理SQL语句时，所有的空格都会被忽略，SQL语句可以写成一行或多行

2.3 检索多个列#

检索多个列，在多个列名之间用逗号隔开（最后一列不能加）

SELECT prod_id,prod_name,prod_price FROM dbo.Products;

2.4 检索所有列#

星号 * 通配符代表所有的列

SELECT * FROM dbo.Products;

2.5 检索不同的值#

distinct 含义：不同的

SELECT DISTINCT vend_id FROM dbo.Products;

2.6 限制结果#

返回指定行数

SELECT TOP 5 prod_name FROM dbo.Products;

2.7 使用注释#

行注释如下：

SELECT prod_name --这是一条注释
FROM dbo.Products;

块注释如下：

/*
 作者：
 联系方式：
 日期：
*/
SELECT prod_name FROM dbo.Products;

第三章 排序检索数据#

3.1 排序数据#

SELECT prod_name FROM dbo.Products ORDER BY prod_name;

注意：应该保证ORDER BY 子句是SELECT语句中最后一条子句。

提示：可以通过非选择列进行排序

3.2 按多个列排序#

多个列排序需要用逗号间隔开

SELECT prod_id,prod_price,prod_name 
FROM dbo.Products 
ORDER BY prod_price,prod_name;

3.3 按列位置排序#

按相对位置序号进行排序

SELECT prod_id,prod_price,prod_name
FROM dbo.Products
ORDER BY 2,3;

提示：不推荐使用此方法，且若排序的列不在SELECT清单中，不能使用。

3.4 指定排序方向#

  • 升序：默认排序方式，关键字：ASC
  • 降序：关键字：DESC
SELECT prod_id,prod_price,prod_name
FROM dbo.Products
ORDER BY prod_price DESC;

多个列排序：

SELECT prod_id,prod_price,prod_name
FROM dbo.Products
ORDER BY prod_price DESC,prod_name;

DESC是DESCENDING的缩写，含义为：下降的，递减的

警告：如果需要对多个列进行降序，每一列都必须指定DESC关键字

第四章 过滤数据#

4.1 使用WHERE子句#

搜索条件也称过滤条件

SELECT prod_name,prod_price
FROM dbo.Products
WHERE prod_price = 3.49;

注意：如果同时出现ORDER BY和WHERE子句时，应该让ORDER BY位于WHERE之后。

4.2 WHERE子句操作符#

操作符 说明
= 等于
<> 不等于
!= 不等于
< 小于
<= 小于等于
!< 不小于
> 大于
>= 大于等于
!> 不大于
BETWEEN 在指定的两个值之间
IS NULL 为NULL值

4.2.1 检查单个值#

价格小于10美元的产品

SELECT prod_name,prod_price
FROM dbo.Products
WHERE prod_price < 10;

4.2.2 不匹配检查#

列出所有不是供应商DLL01制造的产品

SELECT vend_id,prod_name
FROM dbo.Products
WHERE prod_name <> 'DLL01';

提示：单引号用来限定字符串

4.2.3 范围值检查#

检索价格在5美元和10美元之间的所有产品

SELECT prod_name,prod_price
FROM dbo.Products
WHERE prod_price BETWEEN 5 AND 10;

4.2.4 空值检查#

在一个列不包含值时，称其包含空值NULL

检索没有价格的产品

SELECT prod_name
FROM dbo.Products
WHERE prod_price IS NULL;

第五章 高级过滤#

5.1 组合WHERE子句#

操作符：用来连接或改变WHERE子句中的子句的关键字

5.1.1 AND操作符#

要通过不止一个列进行过滤，可以使用AND操作符给WHERE子句附件条件

检索由供应商DLL01制造的价格且价格小于4美元的所有产品

SELECT prod_id,prod_price,prod_name
FROM dbo.Products
WHERE vend_id = 'DLL01' AND prod_price <= 4;

5.1.2 OR操作符#

OR操作符告诉DBMS匹配任意一条件即可

SELECT prod_name,prod_price
FROM dbo.Products
WHERE vend_id = 'DLL01' OR vend_id = 'BRS01';

5.1.3 求值顺序#

当AND操作符和OR操作符同时存在，会有限处理AND操作符

如果要先处理OR操作符，需要对OR操作符用括号

SELECT prod_name,prod_price
FROM dbo.Products
WHERE (vend_id = 'DLL01' OR vend_id = 'BRS01')
AND prod_price >=10;

提示：当AND与OR同时出现时，应该总是使用括号明确的体现优先处理顺序

5.2 IN操作符#

IN操作符用来指定条件范围，范围中的每个条件都可以进行匹配

SELECT prod_name,prod_price
FROM dbo.Products
WHERE vend_id IN ('DLL01','BRS01')
ORDER BY prod_name;

5.3 NOT操作符#

否定其后所跟的任何条件，NOT从不单独使用，它总是与其他操作符一起使用

列出除DLL01之外的所有供应商的产品

SELECT prod_name 
FROM dbo.Products
WHERE NOT vend_id = 'DLL01'
ORDER BY prod_name;

第六章 用通配符进行过滤#

6.1 LIKE操作符#

通配符：用来匹配值的一部分的特殊字符

通配符搜索只能用于文本字段，非文本数据类型字段不能用通配符搜索

6.1.1 百分号（%）通配符#

%表示任何字符出现任意次数

找出所有以词Fish起头的产品

SELECT prod_id,prod_name
FROM dbo.Products
WHERE prod_name LIKE 'Fish%';

说明：搜索是区分大小写的

搜索模式可以同时使用多个通配符

SELECT prod_id,prod_name
FROM dbo.Products
WHERE prod_name LIKE '%bean bag%';

6.1.2 下划线（_）通配符#

下划线（_）只能匹配单个字符

SELECT prod_id,prod_name
FROM dbo.Products
WHERE prod_name LIKE '_ inch teddy bear';

6.1.3 方括号（[ ]）通配符#

找出所有名字以J或M开头的联系人

SELECT cust_contact
FROM dbo.Customers
WHERE cust_contact LIKE '[JM]%'
ORDER BY cust_contact;

6.2 使用通配符的技巧#

  • 不要过度使用通配符
  • 尽量不要把通配符置于开始处
  • 注意通配符的位置

第七章 创建计算字段#

7.1 计算字段#

字段：与列（column）意思相同

字段并不实际存在于数据库表中

尽量选择将转换和格式化工作在数据库服务器完成

7.2 拼接字段#

拼接：将值联结到一起构成单个值

SELECT vend_name +'(' + vend_country + ')'
FROM dbo.Vendors
ORDER BY vend_name;

格式化数据，去掉右边的所有空格

SELECT vend_name +'(' + vend_country + ')'
FROM dbo.Vendors
ORDER BY vend_name;

RTIM函数：去掉字符串左右两边的空格

RTRIM函数：去掉字符串右边的空格

LTRIM函数：去掉字符串左边的空格

一个未命名的列不能用于客户端应用中

别名用AS字段赋予

SELECT RTRIM (vend_name) + '(' + RTRIM(vend_country) + ')'
AS vend_title
FROM Vendors
ORDER BY vend_name;

7.3 执行算术计算#

汇总物品的价格

SELECT prod_id ,quantity,item_price,
       quantity*item_price AS expanded_price
FROM dbo.OrderItems
WHERE order_num = 20008;

SQL算术操作符

操作符 说明
+ 加
- 减
* 乘
/ 除

第八章 使用函数处理数据#

8.1 函数#

SQL函数是不可移植的

8.2 使用函数#

函数的类型

  • 处理文本字符串
  • 数字数据算术操作
  • 处理日期和时间值
  • 返回DBMS正使用的特殊信息的系统函数

8.2.1 文本处理函数#

UPPER( ) 将文本转换为大写

SELECT vend_name,UPPER(vend_name) AS vend_name_upcase
FROM Vendors
ORDER BY vend_name;

8.2.2 日期和时间处理函数#

datepart() 返回年份

SELECT order_num
FROM dbo.Orders
WHERE DATEPART(yy,order_date) = 2012;
