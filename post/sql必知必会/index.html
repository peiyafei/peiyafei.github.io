《SQL必知必会》学习笔记#

第一章 了解SQL#

1.1 数据库的基础#

1.1.1 数据库#

数据库（database）：保存有组织的数据的容器。

DBMS：数据库管理系统

1.1.2 表#

表：某种特定类型数据的结构化清单。

表名：在同一数据库中不能使用相同的表名。

1.1.3 列和数据类型#

表由列组成

列（column）：表中的字段。

数据类型：用于限制列中所存储的数据类型

1.1.4 行#

行（row）：表中的记录

1.1.5 主键#

主键（primary key）：用于唯一表示表中每一行

主键的特点：

  • 唯一性
  • 不允许为NULL值
  • 不允许修改或更新
  • 不能重用（不能删除之后赋予新行）

1.2 什么是SQL#

SQL：Structured Query Language（结构化查询语言）

第二章 检索数据#

2.1 SELECT 语句#

关键字(keyword)：作为SQL组成部分的保留字，不能用作为表或列的名字。

SELECT检索数据，必须至少给出两条信息：选择什么，从什么地方选择

2.2 检索检索单个列#

SELECT prod_name FROM dbo.Products;

提示：

  • 建议SQL语句结尾总是加上分号
  • SQL语句不区分大小写
  • 在处理SQL语句时，所有的空格都会被忽略，SQL语句可以写成一行或多行

2.3 检索多个列#

检索多个列，在多个列名之间用逗号隔开（最后一列不能加）

SELECT prod_id,prod_name,prod_price FROM dbo.Products;

2.4 检索所有列#

星号 * 通配符代表所有的列

SELECT * FROM dbo.Products;

2.5 检索不同的值#

distinct 含义：不同的

SELECT DISTINCT vend_id FROM dbo.Products;

2.6 限制结果#

返回指定行数

SELECT TOP 5 prod_name FROM dbo.Products;

2.7 使用注释#

行注释如下：

SELECT prod_name --这是一条注释
FROM dbo.Products;

块注释如下：

/*
 作者：
 联系方式：
 日期：
*/
SELECT prod_name FROM dbo.Products;

第三章 排序检索数据#

3.1 排序数据#

SELECT prod_name FROM dbo.Products ORDER BY prod_name;

注意：应该保证ORDER BY 子句是SELECT语句中最后一条子句。

提示：可以通过非选择列进行排序

3.2 按多个列排序#

多个列排序需要用逗号间隔开

SELECT prod_id,prod_price,prod_name 
FROM dbo.Products 
ORDER BY prod_price,prod_name;

3.3 按列位置排序#

按相对位置序号进行排序

SELECT prod_id,prod_price,prod_name
FROM dbo.Products
ORDER BY 2,3;

提示：不推荐使用此方法，且若排序的列不在SELECT清单中，不能使用。

3.4 指定排序方向#

  • 升序：默认排序方式，关键字：ASC
  • 降序：关键字：DESC
SELECT prod_id,prod_price,prod_name
FROM dbo.Products
ORDER BY prod_price DESC;

多个列排序：

SELECT prod_id,prod_price,prod_name
FROM dbo.Products
ORDER BY prod_price DESC,prod_name;

DESC是DESCENDING的缩写，含义为：下降的，递减的

警告：如果需要对多个列进行降序，每一列都必须指定DESC关键字
